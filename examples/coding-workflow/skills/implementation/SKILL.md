# Implementation Skill

Use this skill to modify code after the design has been reviewed.

## Goal

Implement the approved design in the bound repository workspace.

## Rules

1. Change only files needed for the issue.
2. Preserve existing project conventions.
3. Prefer small, direct edits over broad rewrites.
4. Do not create the pull request; the workflow owns PR creation.
5. Do not change repository remotes, credentials, or base branch configuration.
6. When validation fails, diagnose the failure and make the smallest correction.

## Output

Return:

```text
## Changes made
## Validation run
## Remaining risks
```
